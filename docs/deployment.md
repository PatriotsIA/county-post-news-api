# Deployment

This service deploys from GitHub through AWS CodePipeline and CodeBuild. It does not use GitHub Actions.

The important detail: this is a TypeScript/SAM app, so CloudFormation cannot deploy the raw `template.yaml` directly. The pipeline must run a build/package step first. CodeBuild turns `template.yaml` into `packaged.yaml`, where Lambda `CodeUri` points to S3.

## Deployment Shape

Use this pipeline shape:

```text
GitHub Source -> CodeBuild -> CloudFormation Deploy
```

The repository files involved are:

- `template.yaml`: SAM template for the Lambda Function URL.
- `buildspec.yml`: CodeBuild commands that install, test, build, run `sam build`, and package the SAM app.
- `packaged.yaml`: generated by CodeBuild and used by the CloudFormation deploy action.

## Before You Start

Use one AWS region consistently for:

- CodePipeline
- CodeBuild
- CodeConnections
- S3 artifact bucket
- CloudFormation stack
- Lambda function

Recommended region: choose the region where CodeBuild quota is available. If one region reports `AccountLimitExceededException` for CodeBuild, use another region or request a CodeBuild quota increase.

Make sure the GitHub repo has this project pushed to the branch you plan to deploy, usually `main`.

## Step 1: Create The Pipeline Source And Build

In AWS Console:

1. Open `Developer Tools > CodePipeline > Pipelines`.
2. Choose `Create pipeline`.
3. On `Choose creation option`, select:
   - Category: `Continuous integration`
   - Template: `CI Build NodeJS`
4. Choose your GitHub source through CodeConnections:
   - Repository: `PatriotsIA/county-post-news-api`
   - Branch: `main`
5. Configure the Node.js build project.
6. Make sure the build uses the repository `buildspec.yml`.

The buildspec runs:

```bash
npm ci
npm run typecheck
npm test
npm run build
sam build
sam package --template-file .aws-sam/build/template.yaml --s3-bucket "$ARTIFACT_BUCKET" --output-template-file packaged.yaml
```

## Step 2: Create Or Choose An Artifact Bucket

CodeBuild needs an S3 bucket for packaged SAM artifacts.

Create or choose a bucket in the same region as the pipeline. Example name:

```text
county-news-api-artifacts-<account-id>-<region>
```

In the CodeBuild project environment variables, add:

```text
ARTIFACT_BUCKET=<your-artifact-bucket-name>
```

The CodeBuild service role must be able to write to this bucket.

## Step 3: Confirm Build Output

The CodeBuild action must output:

```text
packaged.yaml
```

The repo `buildspec.yml` already declares this artifact:

```yaml
artifacts:
  files:
    - packaged.yaml
```

If CodeBuild succeeds, the build artifact should contain `packaged.yaml`.

## Step 4: Add CloudFormation Deploy Stage

After the CI pipeline exists, edit it and add a deploy stage:

1. Open the pipeline.
2. Choose `Edit`.
3. Add a stage after the CodeBuild stage.
4. Add an action:
   - Action provider: `CloudFormation`
   - Action mode: `Create or update a stack`
   - Input artifact: the CodeBuild output artifact
   - Template file: `packaged.yaml`
   - Stack name: `county-news-api`
   - Capabilities: `CAPABILITY_IAM`

Do not point this deploy action at raw `template.yaml`. Raw `template.yaml` has `CodeUri: .`, which causes this error:

```text
CodeUri is not a valid S3 Uri
```

Only deploy `packaged.yaml`.

## Step 5: CloudFormation Permissions

For the first deployment, the CloudFormation deploy role needs permission to create/update:

- Lambda function
- Lambda Function URL
- IAM execution role
- CloudWatch Logs
- S3 deployment artifacts
- CloudFormation stack resources

If the AWS console asks for `CloudFormationResourcePermissions`, this broad first-deploy policy works:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "CloudFormationDeployCountyNewsApi",
      "Effect": "Allow",
      "Action": [
        "cloudformation:*",
        "lambda:*",
        "iam:*",
        "logs:*",
        "s3:*"
      ],
      "Resource": "*"
    }
  ]
}
```

This should be tightened after the first successful deployment.

## Step 6: Run The Pipeline

Run or release a new pipeline change.

Expected result:

```text
Source: success
Build: success
Deploy: success
```

If the build fails, open the CodeBuild logs first.

If deploy fails, open:

```text
CloudFormation > Stacks > county-news-api > Events
```

Look for the first red `CREATE_FAILED` or `UPDATE_FAILED` event. Later rollback messages usually hide the real cause.

## Step 7: Find The API URL

After the CloudFormation deploy succeeds:

1. Open `CloudFormation`.
2. Open the `county-news-api` stack.
3. Go to `Outputs`.
4. Copy `NewsApiUrl`.

The output comes from `template.yaml`:

```yaml
Outputs:
  NewsApiUrl:
    Description: Public Lambda Function URL for the news API.
    Value: !GetAtt NewsApiFunctionUrl.FunctionUrl
```

Test it:

```bash
curl "https://<function-url-id>.lambda-url.<region>.on.aws/health"
curl "https://<function-url-id>.lambda-url.<region>.on.aws/v1/pages/counties/arkansas/polk?limit=48"
```

## Step 8: Configure The Frontend

Set this environment variable in the frontend deployment:

```text
VITE_NEWS_API_URL=https://<function-url-id>.lambda-url.<region>.on.aws/
```

Then redeploy the frontend.

The frontend should use page endpoints for initial loads:

- `/v1/pages/national`
- `/v1/pages/states/:stateSlug`
- `/v1/pages/counties/:stateSlug/:countySlug`

## Troubleshooting

### CodeBuild quota is zero

Error:

```text
Cannot have more than 0 builds in queue for the account
```

Fix:

- Use another region with CodeBuild quota available, or
- Request a CodeBuild quota increase in `Service Quotas`.

### CloudFormation says `CodeUri is not a valid S3 Uri`

Cause:

- CloudFormation is deploying raw `template.yaml`.

Fix:

- Add/run CodeBuild first.
- Deploy CodeBuild output `packaged.yaml`.

### Stack is `ROLLBACK_COMPLETE`

CloudFormation cannot update a stack in `ROLLBACK_COMPLETE`.

Fix:

- Delete the failed stack.
- Re-run the pipeline after fixing the cause.

### No API URL appears

Likely causes:

- Deploy stage did not run.
- Deploy stage failed.
- You are looking at the pipeline infrastructure stack instead of the API app stack.

Fix:

- Open the `county-news-api` application stack, not the `CodePipelineStarterTemplate...` stack.
- Check the `Outputs` tab for `NewsApiUrl`.
